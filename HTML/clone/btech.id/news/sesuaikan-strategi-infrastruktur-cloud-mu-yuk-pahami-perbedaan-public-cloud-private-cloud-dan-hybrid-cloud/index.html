Sesuaikan Strategi Infrastruktur Cloud-mu, Yuk Pahami Perbedaan Public Cloud, Private Cloud, dan Hybrid Cloud

  • 20th Apr 2022
  • Zulfi Al Hakim

Apa Itu Public Cloud?

Public cloud mengacu pada model cloud computing di mana layanan IT dikirimkan melalui internet. Sebagai model paling populer dari layanan cloud computing, public cloud menawarkan banyak pilihan dalam hal solusi dan sumber daya computing untuk memenuhi kebutuhan organisasi atau perusahaan yang terus meningkat.

 

Public cloud menawarkan kemudahan:

  • Elastisitas dan skalabilitas tinggi
  • Biaya relatif rendah karena berbasis langganan
  • Layanan di public cloud mungkin bisa lebih terjangkau, freemium, atau dengan metode langganan, di mana Anda dikenakan biaya berdasarkan apa yang Anda konsumsi.

 

Fungsi dari public cloud ini mulai dari layanan umum seperti email, aplikasi, hingga platform OS tingkat perusahaan atau infrastruktur yang digunakan untuk pengembangan dan pengujian sebuah software.

Nantinya, penyedia public cloud bertanggung jawab untuk mengembangkan, mengelola, dan memelihara kumpulan sumber daya computing yang dibagi ke beberapa penyewa dari seluruh jaringan.

 

Keuntungan menggunakan public cloud

  • Modal yang terjangkau: Investasi yang diperlukan sangatlah minim untuk memelihara infrastruktur IT dalam public cloud.
  • Fleksibilitas teknis: Memiliki skalabilitas dan fleksibilitas tinggi untuk memenuhi tuntutan beban kerja yang kadang tidak terduga. Kapasitas dan sumber daya cloud dengan cepat berkembang demi permintaan pengguna dan tingginya traffic. Pengguna public cloud juga akan mendapatkan availability tinggi karena vendor memiliki sejumlah data center di sebuah area.
  • Anda Bisa fokus pada bisnis: Kerumitan pada masalah IT bisa diminimalisir karena vendor cloud bertanggung jawab atas manajemen infrastruktur, sehingga Anda bisa fokus pada bisnis.
  • Biaya Terjangkau: Harga yang fleksibel berdasarkan penawaran Service Level Agreements (SLA) yang berbeda.
  • Terdapat fitur Monitoring dan Analytics: perusahaan bisa mengumpulkan metrik-metrik berharga pada data yang mereka simpan, atau sumber daya yang mereka gunakan. Public cloud mampu menjalankan data analytic dalam volume tinggi dan mengakomodasi beragam tipe data untuk menarik data insight.

 

Arsitektur public cloud

Public cloud itu sepenuhnya virtual yang bergantung pada jaringan dengan bandwidth tinggi untuk menyalurkan data. Penyedia public cloud memiliki arsitektur multi-tenant yang memungkinkan perusahaan bekerja dengan sebuah shared infrastructure, dan menggunakan sumber daya komputasi serupa. Data-data perusahaan di public cloud terpisah dan tetap terisolasi dari data perusahaan cloud lainnya.

Vendor cloud mengoperasikan layanannya di lokasi terisolasi di dalam daerah public cloud. Lokasi-lokasi ini disebut availability zone yang terdiri dari dua atau lebih data center yang saling terkoneksi. Perusahaan akan memilih zona berdasarkan kedekatan produk mereka dengan pasar masing-masing.

 

Apa Itu Private Cloud?

Private cloud mengacu pada solusi cloud yang didedikasikan untuk digunakan oleh satu organisasi atau perusahaan. Dalam private cloud, Anda tidak berbagi sumber daya cloud computing dengan organisasi atau perusahaan lain.

Pusat data center akan berlokasi dan dioperasikan oleh vendor di luar tempat organisasi atau perusahaan Anda berada. Data center yang jelas akan terisolasi dan hanya akan bisa diakses melalui jaringan pribadi yang aman, dan tidak dibagikan dengan pelanggan vendor yang lain. Private cloud dapat disesuaikan untuk memenuhi kebutuhan dari bisnis yang unik dan spesifik. 

Dengan private cloud, Anda sepenuhnya bertanggung jawab untuk mengatur dan menjaga infrastruktur yang ada. Namun, private cloud juga bisa sepaket dengan pengaturan dari vendor cloud dalam bentuk managed private cloud.

Managed private cloud memperbolehkan perusahaan membuat dan menggunakan sebuah private cloud yang di-deployed, dikonfigurasi, dan diatur oleh vendor. Ini menjadi opsi bagi perusahaan yang kekurangan staf atau mungkin tim IT yang kurang terampil dalam menjaga private cloud. Dengan adanya managed private cloud, perusahaan tetap bisa memiliki private cloud yang bagus tanpa kerumitan sehari-hari dalam mengelola cloud itu sendiri.

 

Keuntungan menggunakan private cloud 

  • Lingkungan yang eksklusif: Lingkungan khusus dan aman yang tidak dapat diakses oleh organisasi atau perusahaan lain.
  • Keamanan bisa di-custom: Kepatuhan terhadap peraturan yang ketat karena organisasi dapat menjalankan protokol, konfigurasi, dan tindakan yang menyesuaikan keamanan berdasarkan persyaratan beban kerja yang unik.
  • Skalabilitas: Private Cloud dapat ditingkatkan kapasitasnya dengan menambah resource hardware yang ada seperti SSD, Memory. 
  • Performa efisien: Private cloud dapat diandalkan untuk performa dan efisiensi SLA yang tinggi.
  • Fleksibilitas: Private cloud sangat fleksibel. Anda bisa mengubah infrastruktur berdasarkan iklim bisnis yang dinamis. Banyak perusahaan yang bergantung pada cloud sebagai disaster recovery, untuk backup data dan aplikasi jika terjadi hal yang tidak diinginkan.

 

Kenapa harus private cloud?

Private cloud adalah solusi ideal untuk pada petinggi IT yang ingin membuat sumber daya perusahaan tersedia on-demand. Ini demi soal keamanan, budget, atau regulasi yang biasa ditemukan di industri finansial atau kesehatan.

Perusahaan-perusahaan di industri tersebut menggunakan protokol enkripsi dan firewall untuk menjaga sistem IT mereka. Namun, private cloud juga akan menambahkan keamanan ekstra karena memiliki akses yang terbatas.

Apakah Anda mau investasi di private cloud atau tidak, itu tergantung pada beban kerja yang ingin di-support. Namun, beban kerja bebas tanpa batas seperti dalam product development, riset, dan telekomunikasi sebaiknya di-support dengan private cloud.

 

Apa Itu Hybrid Cloud?

Sementara itu, hybrid cloud adalah infrastruktur cloud yang menggabungkan public cloud dan private cloud. Contoh mudahnya adalah, sebuah organisasi atau perusahaan dapat menggunakan private cloud untuk IT mereka, lalu melengkapi infrastruktur menggunakan public cloud untuk mengakomodasi lonjakan traffic yang sesekali terjadi.

Bisa juga Anda menggunakan public cloud untuk data yang tidak sensitif guna menghemat biaya, tetapi tetap memiliki private cloud untuk data yang sensitif.

 

Keuntungan menggunakan hybrid cloud

  • Opsi fleksibel: Penerapan berbasis kebijakan yang fleksibel untuk mendistribusikan beban kerja di seluruh lingkungan infrastruktur publik dan pribadi berdasarkan persyaratan keamanan, kinerja, dan biaya.
  • Pertimbangan keamanan: Skalabilitas lingkungan public cloud dicapai tanpa memaparkan beban kerja TI yang sensitif pada risiko keamanan yang melekat.
  • Andal: Mau didistribusikan ke beberapa pusat data, hybrid cloud tetap bisa menghasilkan performa yang optimal.
  • Biaya yang bisa dikendalikan: Data yang sensitif menggunakan private cloud, sementara sisanya bisa disebar ke public cloud yang lebih terjangkau untuk menghemat biaya. 

 

Sebaiknya Pilih Cloud yang Mana?

Pemilihan antara public cloud, private cloud, ataupun hybrid cloud tergantung pada beragam faktor, seperti penggunaan, keterbatasan, dan lain-lain. Namun, kenyataannya ada juga organisasi atau perusahaan yang menggunakan dua atau tiga jenis cloud sekaligus karena kebutuhan yang beragam.

Maka itu, perlu strategi yang matang agar penggunaan tiap jenis cloud bisa optimal sesuai kebutuhan. Bisa dimulai dengan mempertimbangkan beban kerja, jenis data, lalu membuat skala prioritas berdasarkan negatif dan positif dari setiap jenis cloud.
