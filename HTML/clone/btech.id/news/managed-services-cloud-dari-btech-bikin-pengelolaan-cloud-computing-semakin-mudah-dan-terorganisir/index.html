Managed Services Cloud dari Btech bikin Pengelolaan Cloud Computing Semakin Mudah dan Terorganisir

  • 20th Apr 2022
  • Zulfi Al Hakim

Cloud Management perlu disusun dengan strategi yang efisien serta efektif agar Cloud Anda berjalan dengan baik. Apabila Anda merasa bagian dari TIM IT Anda lebih difokuskan kepada pengelolaan dan pengembang di bidang lain (seperti application develop, dsb.), Anda tentu tidak perlu khawatir akan merasa kelabakan dengan task yang banyak. Dengan Btech, strategi dan efektivitas pengelolaan teknologi Cloud Computing Anda dapat berjalan dengan baik.

Sebelum beranjak lebih jauh mengenai keutamaan pengelolaan Cloud Computing atau yang lebih dikenal Managed Services Cloud dari Btech, sebaiknya kita pahami bersama apa itu Cloud Management.

 

Pengertian Cloud Management

Secara detail pengertian Cloud Management adalah pengelolaan data dengan bantuan software yang digunakan untuk merancang, memantau dan mengevaluasi satu atau beberapa aplikasi dan server yang dapat membantu Anda dalam mencapai tujuan strategi efisiensi dan efektivitas pengelolaan cloud computing perusahaan Anda.

Tujuan Cloud Management

Dalam Cloud Management pasti Anda juga menginginkan peningkatan performa, keandalan, efektivitas dan efisiensi biaya dari Cloud Computing. Cloud management juga dapat membantu Anda dalam mencapai tiga tujuan, yaitu:

1. Layanan terpadu / Self-service

Meningkatkan fleksibilitas ketika tim pengelola mengakses Cloud, membuat instance(s) baru, memantau usage dari CPU, RAM, Network Traffic (Bandwidth) dan lainnya serta biaya, yang sudah pasti akan menyesuaikan ke alokasi sumber daya Anda.

2. Otomatisasi Alur / Workflow Automation

Memungkinkan pengelolaan instances di cloud secara otomatis dan tanpa campur tangan manusia.

3. Analisisis / Cloud Analysis

Membantu melihat histori beban kerja / workload yang akan membuat pengalaman analisa Anda menjadi lebih tepat.

 

Step by Step Cloud management

Berikut adalah tahapan yang harus Anda perhatikan dalam melakukan Cloud Management, antara lain:

1. Automation and Orchestration (Planning)

Dalam Perencanaan, Anda harus memetakan kebutuhan infrastruktur seperti spesifikasi server, virtual network, cloud storage, aplikasi dan lain halnya.

2. Security

Tentu Security System ini juga menjadi perhatian yang tidak kalah penting seperti dalam hal Perencanaan. Para pakar IT mengatakan bahwa ancaman serangan siber juga menargetkan pengguna Cloud meskipun tidak sebesar resiko di server tradisional atau on-prem server.

Security System juga sudah terakomodir di dalam Cloud Services ternama lho, seperti Google Cloud Platform, Microsoft Azure, AWS, Alibaba Cloud dan lainnya.

3. Building the Cloud

Ketika kebutuhan virtual server ini sudah Anda dapatkan, maka Anda bisa memulai membangun / build the Cloud dan memigrasikan data yang ada di on-prem server Anda ke Cloud. Tentu hal ini akan berpengaruh kepada kinerja perusahaan Anda terkait dengan persoalan downtime dari proses migrasi server Anda.

4. Testing and Deploying a Service

Setelah layanan teraplikasikan dengan baik, maka barulah server dan aplikasi dapat diakses dan digunakan untuk produktivitas perusahaan.

5. Monitoring, Managing, Metering, Charging, and Optimization

Namun, apakah setelah semuanya itu berjalan, Cloud Management sudah dapat diterapkah?

Tentu saja belum, Anda masih harus melihat usage dari CPU, RAM, Storage, Network, dan lain-lain. Jika terlihat usage dari CPU masih rendah, meskipun Anda berada jam sibuk, maka Anda masih perlu menyesuaikan spesifikasi virtual server Anda atau melakukan scale in / downgrade specification virtual server.

Cloud Management Step by Step

Setelah usage dari komponen virtual server sudah terlihat sesuai, biaya akan mengikuti sesuai dengan usage, maka barulah penerapan Cloud Management dapat berjalan dengan baik.

 

 

 

 

Apa saja Layanan dari Managed Service Cloud Computing?

Mulai dari sini adalah kelebihan Btech dalam jasa yang kami tawarkan kepada Anda, layanan yang kami berikan adalah sebagai berikut:

 

1. Cloud Assessment and Planning

Langkah pertama yang instruktur dan engineer Btech lakukan adalah menilai dan merencanakan seberapa siap pengembangan Anda atau workload Anda yang siap dipindahkan ke teknologi Cloud. Dengan begitu, dapat TIM kami mulai melalui tahap-tahap

• Assess : Menilai lingkungan, infrastruktur, workload, dan aplikasi. Hal ini memungkinkan perusahaan Anda untuk menentukan aplikasi dan data dapat dipindahkan atau tidak dapat dengan mudah dipindahkan ke lingkungan cloud dan model pengiriman apa (Public, Private, atau Hybrid) yang dapat dijalankan.

• Plan : Memetakan infrastruktur berdasarkan kebutuhan dan kinerja aplikasi.

• Visualize : Mengelompokkan aplikasi berdasarkan interkonektivitas dan dependensi, wawasan yang diperoleh dari Assess dan pemetaan infrastruktur.

• Prepare : Menentukan kelayakan, kesiapan, dan biaya migrasi aplikasi Anda.

• Estimate : memperkirakan Total Biaya Kepemilikan (TCO) dari workload aplikasi di on-prem server dengan di Cloud.

 

2. Cloud Migration or Deployment

Proses migrasi:

• Mencari histori penggunaan pemakaian on-prem (VM, jaringan, aplikasi, dll).

• Memetakan dependensi dan menentukan topologi jaringan.

• Merekomendasikan pengoptimalan untuk workload.

• Tentukan model layanan komputasi awan yang tepat.

• Menyediakan rencana dan strategi migrasi cloud (decommission / lift-and-shift / re-platform / re-architect / new deployments).

 

3. Infrastructure Management

• Configuration Management: menjaga perangkat lunak tetap up-to-date, patching, mengatur ulang kata sandi, dan mengatur ukuran sumber daya

• Backup/ Disaster Recovery : proses backup untuk semua aplikasi, workload, dan data. DR untuk aplikasi penting misi dengan SLA, RPO, RTO yang sangat baik

• Identity Management: mengelola akun pengguna, single sign-on, memberikan akses berbasis role.

 

4. Cloud Monitoring Service

Jenis Cloud Monitoring:

• System Health Monitoring : Pemantauan secara menyeluruh ke VM, penggunaan CPU, penggunaan RAM, penyimpanan, dan kinerja OS.

• Log analytics / Alerting : Menganalisis log mendalam tentang performa, security, resource comsumption, dan sejumlah metrik penting lainnya.

• Database Monitoring : Kami membantu memastikan ketersediaan database server.

• Application Performance Monitoring : End-to-end Tracking dari semua aspek aplikasi (atau halaman web).

 

5. Cost Optimization

Proses optimasi:

• Usage Visibility : Memungkinkan Anda untuk mendapatkan visibilitas penggunaan berdasarkan environment, workload, aplikasi, atau end-user.

• Cost Control : Membantu Anda mengidentifikasi beberapa hal inefisiensi dan redundansi.

• Billing Management : Tampilan biaya Cloud yang disederhanakan sehingga memudahkan dalam pembacaan tagihan.

• Resource Utilization : Memastikan bahwa resources disediakan dengan benar.

6. Support

Kami sangat berhati-hati dan mendukung saat Anda dalam menangani kendala, pelanggaran, inefisiensi, dan skenario bencana, dengan memberikan:

• Supporting 7×24 dengan tim lokal.

• Didukung dengan leveling Support mulai dari Customer Service – Advanced Troubleshooting – and Specialized Architects.

• Waktu menanggapi masalah Anda dengan cepat: kurang dari 24 jam.

• Dedicated Support – Technical Account Manager / Architect.
